# OVP Grammar — Project Guide for Claude

## What This Is

A formal grammar-based OVP (Owens Valley Paiute) ↔ English translator, built as a proof of concept to demonstrate that ANTLR4 + rule-based morphology outperforms the JSON-table + LLM approach in Jared Coleman's system. See `report.md` for the full analysis.

## Key Commands

```bash
make build      # Build Docker image (Java JRE + ANTLR4 4.13.1 + Python runtime)
make generate   # Recompile grammar/OVP.g4 -> src/generated/*.py  (run after editing .g4)
make test       # Run all 20 pytest tests inside Docker
make shell      # Interactive bash shell inside container
make clean      # Remove generated files
```

All commands run inside Docker — Java is not installed on the host machine.

## Current Status (as of April 2026)

**OVP → English: complete.**
- All 4 sentence types: NounSV, NounSVO, PronounVS, PronounOVS
- All 6 tenses: past (ku), present (dü), ongoing (ti), future (wei), perfect (pü), going-to (gaa-wei)
- Correct English subject-verb agreement (I am / he is / we are, etc.)
- Lenition handled (düka=eat, buni=see, zawa=cook, etc.)
- Proximal/distal determiners (this/that)
- Grammar enforcement: SyntaxError on invalid OVP input
- 20/20 tests passing

**English → OVP: not yet implemented.**
- OVP morphology generation rules are in `ovp-lang/language.py` (Jared's repo)
- English parsing is the hard part — LLM normalization to structured SVO is appropriate here
- OVP generation from structured SVO is straightforward and rule-based

## Important Technical Notes

### Grammar file (OVP.g4)
- ANTLR4 uses single-quoted string literals only (not double quotes)
- Apostrophes (glottal stops) inside strings must be escaped: `'isha\''`
- Token ordering matters: longer tokens before shorter (maximal munch)
  - `neika` before `eika`, `noka` before `oka`, `uu` before `u`
  - `GAA_WEI` (`gaa-wei`) must precede `HYPHEN` since it contains a hyphen
- After editing OVP.g4, always run `make generate` before `make test`

### Docker volume mount
- `docker-compose.yml` mounts `./src`, `./tests`, `./grammar` as volumes
- The `make generate` target uses a targeted volume mount (`-v $(PWD)/src/generated:/app/src/generated`) to write generated files back to the host
- Do NOT use `./src:/app/src` as a single mount — it hides the build-time generated files

### Lexicon (lexicon.py)
- `PRONOUN_PERSON` maps OVP pronouns to grammatical person (1sg, 3sg, 1pl, etc.)
- `conjugate(verb, tense, person)` handles all tense + agreement combinations
- Both base and lenited verb forms map to same English (e.g., `"düka": "eat"`)
- `GLOTTAL_STOP_NOUNS` determines short vs long object suffix (-eika/-oka vs -neika/-noka)

### Translator (translator.py)
- `OVPToEnglish` extends the ANTLR4-generated `OVPVisitor`
- `self._person` is set at the sentence level and read by verb visitors
- `StrictErrorListener` converts ANTLR4 parse errors to Python `SyntaxError`
